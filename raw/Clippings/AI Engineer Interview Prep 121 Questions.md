### Round 3: AI Engineer Technical Concepts:

1. What is the bias-variance tradeoff?

Bias is the error from wrong assumptions in the model — high bias means the model underfits and misses patterns. Variance is the error from sensitivity to small fluctuations in training data — high variance means the model overfits and memorizes noise. The tradeoff: reducing bias tends to increase variance and vice versa. The goal is to find the sweet spot that minimizes total error on unseen data. Regularization, cross-validation, and ensemble methods help manage this balance.

---

1. What is overfitting and how do you prevent it?

Overfitting is when a model learns the training data too well — including noise — and fails to generalize to new data. Signs: high training accuracy, low validation accuracy.

Prevention techniques:

- Regularization (L1/L2) — penalizes large weights
- Dropout — randomly disables neurons during training
- Early stopping — halt training when validation loss starts increasing
- Data augmentation — artificially expand training data
- Cross-validation — use k-fold to evaluate generalization
- Reduce model complexity — fewer layers/parameters
- More training data

---

1. Explain the difference between L1 and L2 regularization.

L1 (Lasso): Adds the sum of absolute values of weights to the loss. Tends to produce sparse models — drives some weights to exactly zero, effectively doing feature selection. Good when you suspect many features are irrelevant.

L2 (Ridge): Adds the sum of squared weights to the loss. Distributes penalty across all weights, shrinking them but rarely to zero. Good when all features contribute and you want smoother models.

Key difference: L1 produces sparsity (feature selection), L2 produces small but non-zero weights (shrinkage).

---

1. What is gradient descent and what are its variants?

Gradient descent is an optimization algorithm that iteratively adjusts model parameters in the direction that minimizes the loss function, using the gradient (derivative) of the loss.

Variants:

- Batch Gradient Descent: Uses the entire dataset per update. Stable convergence but slow on large datasets.
- Stochastic Gradient Descent (SGD): Uses one sample per update. Fast but noisy — can escape local minima.
- Mini-batch Gradient Descent: Uses small batches (e.g., 32–256 samples). Best of both — most commonly used in practice.
- Adam: Adaptive learning rates per parameter using first and second moment estimates. Most popular optimizer today.
- RMSProp: Adapts learning rate based on recent gradient magnitudes. Good for RNNs.

---

1. What is cross-validation and why do we use it?

Cross-validation is a technique to evaluate model generalization on unseen data by splitting the dataset into multiple train/validation folds.

K-Fold CV: Data is split into k equal folds. The model trains on k-1 folds and validates on the remaining fold. This repeats k times, and results are averaged.

Why use it: Reduces the impact of how the data is split, gives a more reliable estimate of model performance, and helps detect overfitting. Stratified k-fold ensures class balance is maintained across folds — important for imbalanced datasets.

---

1. Explain precision, recall, F1 score and when to use each.

Precision = TP / (TP + FP) — Of all predicted positives, how many were actually positive. Use when false positives are costly (e.g., spam detection — you don’t want to flag legitimate emails as spam).

Recall = TP / (TP + FN) — Of all actual positives, how many did we catch. Use when false negatives are costly (e.g., cancer detection — missing a real case is dangerous).

F1 Score = 2 \* (Precision \* Recall) / (Precision + Recall) — Harmonic mean of both. Use when you need a balance, especially with imbalanced classes.

AUC-ROC is threshold-independent and measures overall model discrimination ability — better for comparing models.

---

1. How do you handle imbalanced datasets?
- Resampling: Oversample the minority class (SMOTE) or undersample the majority class
- Class weights: Pass class\_weight=’balanced’ to penalize misclassifying the minority class more
- Use appropriate metrics: Avoid accuracy — use F1, AUC-ROC, or precision-recall curve
- Ensemble methods: BalancedRandomForest, EasyEnsemble
- Threshold tuning: Adjust the decision threshold rather than defaulting to 0.5
- Collect more data for the minority class if possible

---

1. What is the difference between bagging and boosting?

Bagging (Bootstrap Aggregating): Trains multiple models in parallel on random subsets of data. Combines predictions via voting (classification) or averaging (regression). Reduces variance. Example: Random Forest.

Boosting: Trains models sequentially — each new model focuses on errors made by the previous one. Combines predictions via weighted voting. Reduces bias. Examples: AdaBoost, XGBoost, LightGBM.

Key difference: Bagging = parallel, reduces variance. Boosting = sequential, reduces bias. Boosting generally achieves higher accuracy but is more prone to overfitting noisy data.

---

1. What is the curse of dimensionality?

As the number of features (dimensions) increases, the data becomes increasingly sparse in the feature space. This makes it harder for models to find meaningful patterns because:

- Distance metrics become less meaningful (all points are roughly equidistant)
- Exponentially more data is needed to maintain the same density
- Models overfit more easily

Solutions: Dimensionality reduction (PCA, t-SNE, UMAP), feature selection, regularization.

---

1. What is PCA and when would you use it?

Principal Component Analysis (PCA) is an unsupervised dimensionality reduction technique. It projects data onto a new set of orthogonal axes (principal components) that capture the maximum variance in the data.

Steps: Standardize → compute covariance matrix → compute eigenvectors/eigenvalues → project onto top-k components.

Use when: Features are highly correlated, you want to reduce computational cost, visualize high-dimensional data (2D/3D), or remove noise before modeling.

Limitation: PCA creates new features that are linear combinations of originals — interpretability is lost.

---

1. Explain the Transformer architecture and self-attention.

The Transformer is the foundation of modern LLMs (BERT, GPT, Gemini, Llama). It replaces recurrence with self-attention, enabling parallelization.

Self-attention computes how much each token should “attend” to every other token in the sequence:

- Query (Q), Key (K), Value (V) matrices are learned projections of the input
- Attention score = softmax(QK^T / sqrt(d\_k)) \* V
- This lets the model weigh relevant context regardless of position distance

Multi-head attention runs self-attention in parallel across multiple “heads,” each learning different relationship patterns. The Transformer also uses positional encodings (since it has no built-in sequence order), feed-forward layers, layer normalization, and residual connections.

Why it matters: Self-attention is O(n²) in sequence length — this is the scalability bottleneck for long-context models.

---

1. What is the vanishing/exploding gradient problem?

Vanishing gradients: In deep networks, gradients shrink exponentially as they backpropagate through layers. Weights in early layers barely update, making deep networks hard to train. Common with sigmoid/tanh activations.

Exploding gradients: Gradients grow exponentially — weights update wildly, causing instability.

Solutions:

- ReLU and variants (Leaky ReLU, GELU) — don’t saturate, reduce vanishing
- Batch normalization — normalizes activations, stabilizes gradients
- Residual connections (ResNets) — skip connections allow gradients to flow directly
- Gradient clipping — cap gradients to a max norm (fixes exploding)
- Careful weight initialization (Xavier, He)
- LSTMs/GRUs — designed with gating mechanisms to handle long-range dependencies

---

1. What is the difference between CNN, RNN, and Transformer?

CNN (Convolutional Neural Network): Captures local spatial patterns via convolution filters. Best for: image classification, object detection, spatial data. Fixed receptive field — doesn’t naturally handle sequential dependencies.

RNN (Recurrent Neural Network): Processes sequences step-by-step, passing hidden state forward. Best for: time series, text (historically). Problem: vanishing gradients over long sequences; slow due to sequential computation.

Transformer: Uses self-attention to model all pairwise token relationships in parallel. Best for: NLP, multimodal tasks, long sequences. Scales well but quadratic memory in sequence length.

Today: Transformers have largely replaced RNNs for most NLP and sequence tasks.

---

1. What is dropout and how does it work?

Dropout randomly sets a fraction of neuron outputs to zero during training (e.g., 20–50% of neurons per forward pass). This forces the network to learn redundant representations — no single neuron can be relied upon exclusively.

Effect: Acts as an ensemble of many sub-networks. Regularizes the model, reducing overfitting.

At inference time, dropout is turned off and all neurons are active — but weights are scaled by the dropout probability to maintain expected activation values.

---

1. Explain batch normalization. Why is it used?

Batch normalization normalizes the activations of each layer across a mini-batch to have mean 0 and variance 1, then applies learnable scale (gamma) and shift (beta) parameters.

Benefits:

- Stabilizes and accelerates training by reducing internal covariate shift
- Acts as mild regularization, reducing the need for dropout
- Allows use of higher learning rates
- Makes networks less sensitive to weight initialization

Applied typically before or after the activation function. Note: BN behaves differently at training vs. inference — at inference it uses running statistics from training.

---

1. What is the difference between fine-tuning and RAG (Retrieval-Augmented Generation)?

Fine-tuning: Updates a pre-trained model’s weights on a domain-specific dataset. The knowledge is baked into the model. Best when: you need consistent tone/style, domain-specific reasoning, or offline deployment. Expensive to update — requires retraining when knowledge changes.

RAG: Keeps the base model frozen. At inference, retrieves relevant documents from an external knowledge base and injects them into the prompt as context. Best when: knowledge needs to stay current, you want source attribution, or the dataset is too large to fine-tune on. More transparent and easier to update.

Rule of thumb: Use RAG for dynamic, frequently-updating knowledge. Use fine-tuning for style, task format, or specialized reasoning that won’t change.

---

1. What causes hallucinations in LLMs and how do you mitigate them?

Hallucinations occur when an LLM generates text that is confidently stated but factually incorrect. Causes: training data noise, insufficient grounding, model’s tendency to complete patterns even without factual basis.

Mitigation strategies:

- RAG: Ground responses in retrieved source documents
- Temperature reduction: Lower temperature makes outputs more deterministic
- Chain-of-thought prompting: Encourages step-by-step reasoning before conclusions
- Fact-checking layers: Post-generation verification against a knowledge base
- Confidence calibration: Train models to express uncertainty
- Instruction fine-tuning: Train models to say “I don’t know” when uncertain
- Human feedback (RLHF): Penalize hallucinated outputs during training

### ML SYSTEM DESIGN CONCEPTS

1. How would you design a recommendation system?

Key components:

1. Data layer: User interaction logs (clicks, purchases, watch time), item metadata, user profiles
2. Candidate generation: Retrieve top-N candidates efficiently using collaborative filtering (matrix factorization, ALS) or embedding-based nearest neighbor search (FAISS, Annoy)
3. Ranking: Re-rank candidates using a more complex model (gradient boosted trees, two-tower neural net) trained on engagement signals
4. Serving: Low-latency inference with feature stores for pre-computed embeddings
5. Monitoring: Track CTR, conversion, diversity, and freshness metrics
6. Retraining: Trigger when distribution shift or performance degradation detected

Two main approaches:

- Collaborative filtering: Recommendations based on user behavior patterns (users who liked X also liked Y)
- Content-based filtering: Recommendations based on item features matching user preferences
- Hybrid systems combine both

---

1. How do you detect and handle model drift in production?

Data drift: Input feature distribution shifts from training distribution. Detect via: Population Stability Index (PSI), KL divergence, statistical tests (KS test) on feature distributions.

Concept drift: Relationship between inputs and outputs changes (e.g., user behavior after a major event). Detect via: monitoring prediction confidence, tracking ground truth labels when available, business metric degradation.

Handling drift:

- Set automated alerts when drift metrics exceed thresholds
- Trigger retraining pipelines on a schedule or on alert
- Use online learning for models requiring continuous adaptation
- Maintain shadow models trained on recent data as fallback
- Version all models and data pipelines for rollback

---

1. What is the difference between online learning and batch learning?

Batch learning: Model is trained on a fixed dataset, deployed, and retrained periodically. Simple and stable but lags behind real-time changes. Best when data is static or changes slowly.

Online learning: Model updates continuously as new data arrives. Can adapt to real-time changes. Examples: Stochastic gradient descent on streaming data, Vowpal Wabbit. Best for high-velocity data streams like ad click prediction or fraud detection.

Trade-off: Online learning is harder to monitor, debug, and roll back — batch learning is safer but less responsive.

### GENERATIVE AI / LLMs

1. What is the difference between GPT (decoder-only) and BERT (encoder-only)?

BERT (encoder-only): Processes the full sequence bidirectionally — every token attends to all other tokens. Trained via masked language modeling (MLM) and next sentence prediction. Best for: text classification, NER, question answering, embeddings.

GPT (decoder-only): Processes tokens left-to-right (causal masking — each token only attends to previous tokens). Trained via next token prediction (autoregressive). Best for: text generation, summarization, code generation, chat.

Encoder-decoder (T5, BART): Uses both — encoder processes input, decoder generates output. Best for: translation, summarization with a defined input-output structure.

---

What is RLHF (Reinforcement Learning from Human Feedback)?

RLHF is the technique used to align LLMs to human preferences. Three stages:

1. Supervised Fine-Tuning (SFT): Fine-tune a pre-trained model on high-quality human-written demonstrations.
2. Reward Model Training: Human annotators rank model outputs. A reward model is trained to predict these rankings — essentially learning “what humans prefer.”
3. RL Optimization: The language model is fine-tuned using PPO (Proximal Policy Optimization) to maximize the reward model’s score while staying close to the original SFT model (via KL penalty to prevent reward hacking).

Used in: ChatGPT, Claude, Gemini, Llama-based instruction models. Key challenge: reward hacking — the model learns to game the reward model rather than genuinely improve.

---

What is RAG and how does it work end to end?

RAG (Retrieval-Augmented Generation) augments an LLM with an external knowledge base at inference time.

Pipeline:

1. Indexing: Documents are chunked → embedded using an embedding model → stored in a vector database (Pinecone, FAISS, Weaviate)
2. Retrieval: User query is embedded → nearest neighbor search retrieves top-k relevant chunks
3. Augmentation: Retrieved chunks are injected into the prompt as context
4. Generation: LLM generates a response grounded in the retrieved context

Key design decisions: chunk size, embedding model choice, retrieval strategy (dense vs sparse vs hybrid BM25+dense), re-ranking (cross-encoder), context window management.

---

What is the difference between embeddings and one-hot encoding?

One-hot encoding: Represents categorical data as a binary vector. Sparse (most values 0), no semantic relationships captured. Vector for “cat” and “dog” are equally distant from each other and from “animal.”

Embeddings: Dense, low-dimensional learned representations. Semantically similar items are close in the embedding space. Capture relationships: “king - man + woman ≈ queen.” Efficient — much smaller than one-hot vectors. Learned during training (Word2Vec, GloVe, Transformer embeddings).

When to use: One-hot for simple categorical variables in classical ML. Embeddings for text, high-cardinality categoricals, and neural networks.

---

What is prompt engineering and what are the key techniques?

Prompt engineering is the practice of crafting input prompts to elicit desired behavior from LLMs without modifying model weights.

Key techniques:

- Zero-shot: Give the task instruction with no examples. “Classify this review as positive or negative:...”
- Few-shot: Provide 2–5 examples in the prompt before the real query. Dramatically improves performance on structured tasks.
- Chain-of-thought (CoT): Ask the model to reason step by step (”Let’s think step by step”). Improves performance on math and logic tasks.
- ReAct: Combine reasoning and tool use — model thinks, then acts (calls a tool), then observes results.
- System prompts: Define the model’s role and constraints at the start of the conversation.
- Self-consistency: Sample multiple CoT paths and take the majority answer.

### MLOPS & PRODUCTION

1. What is a feature store and why is it important?

A feature store is a centralized repository for storing, versioning, and serving ML features — both for training and real-time inference.

Problems it solves:

- Training-serving skew: Features computed differently in training vs. production
- Feature duplication: Teams recomputing the same features independently
- Latency: Pre-computing and caching features for low-latency serving
- Consistency: Ensures the same feature logic is used in all models

Components: Offline store (data warehouse for training), online store (low-latency key-value store for serving), feature pipelines, feature registry (metadata and versioning).

Examples: Feast (open-source), Tecton, Databricks Feature Store, Vertex AI Feature Store.

---

1. What is the difference between model parallelism and data parallelism?

Data parallelism: Split the training data across multiple GPUs/nodes. Each device holds a full copy of the model and processes a different mini-batch. Gradients are averaged and synchronized across devices. Most common approach — used in standard distributed training (PyTorch DDP).

Model parallelism: Split the model itself across multiple devices — different layers run on different GPUs. Used when the model is too large to fit on a single GPU. Examples: pipeline parallelism (split by layers), tensor parallelism (split weight matrices).

In practice: Large LLMs use both — data parallelism for batch processing and model parallelism to fit massive models across GPU clusters.

---

1. What are the key metrics for evaluating a recommendation system?

Offline metrics (during training/evaluation):

- Precision@K: Of the top-K recommendations, how many are relevant?
- Recall@K: Of all relevant items, how many appear in the top-K?
- NDCG (Normalized Discounted Cumulative Gain): Measures ranking quality — rewards relevant items appearing higher in the list
- Mean Average Precision (MAP): Average precision across all users

Online metrics (A/B tests in production):

- Click-through rate (CTR): Percentage of recommendations clicked
- Conversion rate: Percentage that lead to purchase/action
- Dwell time / watch time: Engagement depth
- Diversity and novelty: Avoids filter bubbles

Business metrics: Revenue per user, retention, churn reduction.

---

1. What is A/B testing and how do you design one for an ML model?

A/B testing is a controlled experiment where traffic is split between a control group (model A, baseline) and a treatment group (model B, new model) to measure the causal impact of the change.

Design steps:

1. Define hypothesis and primary metric (e.g., CTR, revenue, retention)
2. Determine sample size using power analysis (desired significance level α, power 1-β, expected effect size)
3. Randomly assign users to control/treatment — ensure no leakage
4. Run for sufficient duration to capture weekly cycles and avoid novelty effects
5. Analyze results: t-test or z-test for significance; check p-value < 0.05
6. Check guardrail metrics: ensure secondary metrics (latency, errors) don’t degrade

Common pitfalls: Stopping early (peeking problem), multiple testing without correction, Simpson’s paradox in segmented analysis.

---

1. How do you reduce model inference latency in production?

Key strategies:

- Model quantization: Reduce weight precision from FP32 → INT8 or FP16. Significantly reduces model size and speeds up inference with minimal accuracy loss.
- Model pruning: Remove low-importance weights or neurons. Reduces model size.
- Knowledge distillation: Train a smaller “student” model to mimic a larger “teacher” model. Maintains accuracy with faster inference.
- ONNX export: Export model to ONNX format for optimized cross-platform inference.
- Caching: Cache frequent predictions or embeddings in a key-value store (Redis).
- Batching: Batch multiple inference requests together to maximize GPU utilization.
- Async serving: Use async endpoints with request queuing to handle spikes.
- Hardware acceleration: Deploy on GPUs, TPUs, or specialized inference chips (NVIDIA TensorRT, AWS Inferentia).

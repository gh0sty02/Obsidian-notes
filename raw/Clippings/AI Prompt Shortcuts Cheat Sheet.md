**100+ AI Prompt Shortcuts That Make Claude & ChatGPT 10x Better**

*Add any of these to the start of your message and watch the output transform.*

1. **ELI5:** — Explain like I’m 5, zero jargon
2. **ELI10:** — Slightly more detail, still beginner-friendly
3. **TLDR:** — 1–2 sentence summary only
4. **BLUF:** — Most important point in sentence 1
5. **SHORT:** — 3 sentences max, no exceptions
6. **MINI:** — One sentence only
7. **LONG:** — Exhaustive answer, don’t hold back
8. **DENSE:** — Maximum information, bullets, no filler
9. **SIMPLE:** — Plain language, no technical terms
10. **FORMAL:** — Polished, professional prose
11. **CASUAL:** — Like texting a smart friend
12. **TWEET:** — Compress into 280 characters
13. **THREAD:** — Format as a 5–8 post Twitter/X thread
14. **MEMO:** — Format as a professional internal memo
15. **BULLETS:** — Bullet points only, no intro or conclusion
16. **TABLE:** — Format as a markdown table
17. **JSON:** — Output valid JSON only, no prose
18. **XML:** — Output valid XML only, no prose
19. **CODE:** — Output only code, no explanation
20. **NOCODE:** — No code blocks, explain conceptually only
21. **ULTRA:** — Maximum depth reasoning before answering
22. **STEP:** — Force step-by-step reasoning, show all steps
23. **WHY5:** — Apply 5 Whys root-cause analysis
24. **INVERT:** — How to guarantee failure, then flip it
25. **STEEL:** — Steelman — give the argument its strongest version
26. **DEVIL:** — Argue the opposite position hard
27. **BAYES:** — Apply Bayesian reasoning with probability updates
28. **2ND:** — Analyze second and third-order effects only
29. **FIRST:** — First-principles decomposition from scratch
30. **OODA:** — Observe-Orient-Decide-Act cycle before answering
31. **TREE:** — Explore 3 branches of thought then converge
32. **PRE:** — Pre-mortem — assume it failed, work backwards
33. **POST:** — Post-mortem — what went wrong and why
34. **ASSUME:** — List all hidden assumptions before answering
35. **COUNTER:** — Generate the strongest counterargument to my point
36. **CONTRAST:** — Force explicit comparison of two options
37. **2ND\_FX:** — Map cascade effects two levels deep
38. **SCOPE:** — Define and constrain the exact problem space first
39. **ZOOM:** — Alternate between tactical detail and strategic altitude
40. **KNOWN:** — Map known, known-unknowns, and unknown-unknowns
41. **BRUTAL:** — Brutal honesty, no softening, no compliment sandwich
42. **BLUNT:** — No hedging, no caveats, no disclaimers. Just the answer
43. **CITE:** — Back every claim with a source or flag as unverified
44. **CONF:** — Add confidence % in brackets after every factual claim
45. **FLAG:** — Explicitly flag every assumption and uncertainty
46. **PUSH:** — Push back on me if I’m wrong, don’t just agree
47. **NOPAD:** — No filler, no greetings, no “great question”
48. **NOLIST:** — Prose only, no bullet points or numbered lists
49. **NOHYPE:** — Strip all marketing language and buzzwords
50. **DUAL:** — Dual newspaper test — not harmful, not needlessly unhelpful
51. **TEACH:** — Explain pedagogically, build from fundamentals up
52. **QUIZ:** — End the explanation with 3 quiz questions
53. **GAP:** — Identify gaps in my understanding from what I wrote
54. **SOCRATIC:** — Only respond with questions to guide me to the answer
55. **FEYNMAN:** — Explain using only simple analogies and plain words
56. **ANALOGIES:** — Give 5 analogies from different domains
57. **ANKI:** — Generate 5 Anki flashcard Q&A pairs on this topic
58. **MAP:** — Create a concept map or knowledge structure
59. **MISCONC:** — List common misconceptions people have about this
60. **HISTORY:** — Explain the historical origin and evolution of this idea
61. **PLAN:** — Step-by-step action plan with owners and deadlines
62. **SPRINT:** — Break into a 2-week sprint with daily tasks
63. **FAST:** — Fastest path to done, skip theory, just practical steps
64. **BLOCK:** — Turn this into a time-blocked daily schedule
65. **PRIOR:** — Prioritize using the Eisenhower matrix
66. **80/20:** — Identify the 20% of effort driving 80% of results
67. **CHECKLIST:** — Convert this into an actionable checklist
68. **BATCH:** — Group tasks by context for batched execution
69. **EXIT:** — Define clear, measurable exit criteria for this task
70. **DELEGATE:** — Identify what can be delegated and to whom
71. **DRAFT:** — Rough first draft, no polish needed
72. **POLISH:** — Improve flow, word choice, and clarity
73. **HOOK:** — Write only the opening hook, make it irresistible
74. **CTA:** — Write a compelling call-to-action
75. **REFRAME:** — Reframe the message to land better with the audience
76. **COMPRESS:** — Cut to 50% length without losing meaning
77. **EXPAND:** — Expand into a fuller version with more detail
78. **APHORISM:** — Distill the core idea into one quotable maxim
79. **TONE:** — Shift the tone while keeping the meaning intact
80. **ANTIJAR:** — Remove all jargon, plain English only
81. **PITCH:** — Frame as an investor pitch narrative
82. **PERSONA:** — Build a detailed user persona
83. **MOAT:** — Analyze competitive moat and defensibility
84. **WORKING\_BACK:** — Amazon working-backwards from the press release
85. **SCENARIO:** — Generate 3–4 distinct plausible future scenarios
86. **SWOT:** — Structured SWOT analysis, specific and non-generic
87. **PRD:** — Generate a Product Requirements Document
88. **JTBD:** — Apply Jobs-To-Be-Done framework
89. **OKR:** — Generate Objectives and Key Results
90. **RICE:** — Prioritize using the RICE scoring framework
91. **DELPHI:** — Generate → self-critique → refine, show all 3 versions
92. **V2:** — Generate a significantly improved second version
93. **ALT3:** — Give 3 completely different alternatives
94. **REMIX:** — Remix content for a different audience or format
95. **NEXT:** — What are the 3 best next questions I should ask?
96. **CHAIN:** — Design a prompt chain for this multi-step task
97. **RECAP:** — Summarize conversation into key decisions and open questions
98. **COMPRESS\_CTX:** — Compress conversation into a dense context card
99. **AUDIT:** — Code review — readability, performance, security, style
100. **REFACTOR:** — Refactor for clarity and idiomatic style
101. **TESTS:** — Write comprehensive unit tests for this code
102. **BUG:** — Debug this — explain what’s wrong and why
103. **DOCS:** — Write documentation or docstrings for this code
104. **SEC:** — Security audit — find vulnerabilities, rate by severity
